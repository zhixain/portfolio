This project implements a standalone face recognition system on the PYNQ-Z1 embedded platform, combining MobileFaceNet CNN inference with FPGA hardware acceleration. The system operates entirely offline without cloud dependency, covering face registration, feature extraction, and identity verification via cosine similarity.

System Architecture

The system divides responsibilities between:

• PS (ARM Cortex-A9)
  - Camera capture & preprocessing
  - Overlay loading & system control
  - Feature vector storage & comparison

• PL (FPGA Fabric)
  - Conv/BN/PReLU/Linear HLS IP cores
  - AXI4-Lite: control registers
  - AXI4-Stream: data pipeline
  - AXI DMA: bulk data transfer

PS and PL communicate via AXI Interconnect, with DMA reducing CPU overhead during large-scale feature map transfers.

HLS Optimization Strategy

Two key techniques applied to compute layers:

① Loop Pipelining

Interleaves different loop iterations within a single clock cycle.
→ Achieved Initiation Interval (II) = 1

② Array Partitioning

Splits arrays into independently accessible blocks, reducing memory access congestion.

Face Detector Selection: Haar vs. MTCNN

✓ Haar Cascade Classifier

  • Fast inference on ARM CPU
  • Low computational overhead
  • Selected for real-time capture
Trade-off: May miss faces at extreme angles.

✗ MTCNN

  • Higher recall & precision
  • Precision: 0.9182
  • Recall: 0.8656
Trade-off: Latency too high for embedded real-time.
